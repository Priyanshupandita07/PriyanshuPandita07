<details>
<summary><strong>🔷 Chicago Crime Intelligence Assistant</strong> &nbsp;—&nbsp; RAG Pipeline on BigQuery × Gemini</summary>
<br/>
A production-grade Retrieval-Augmented Generation system built over 500K+ rows of Chicago crime data hosted in BigQuery. The system accepts natural language queries, retrieves semantically relevant crime records, and synthesises analytical responses using Gemini 1.5. Designed as a domain-specific intelligence layer over a large-scale public dataset, demonstrating how LLMs can be grounded in structured data warehouses for enterprise Q&A applications.

<br/>
AttributeDetailStackPython · BigQuery · Gemini API · PySpark · Cloud Storage · Dataproc ServerlessScale500,000+ crime records · Automated nightly ingestion via Cloud SchedulerPerformanceServerless Spark execution · Partitioned BigQuery tables · Optimised SQL retrievalArchitectureRAG loop: query → BigQuery retrieval → Gemini context injection → structured responseImpactDemonstrates LLM grounding over structured warehouse data — portfolio-differentiating for DE + AI rolesRepositorygithub.com/Priyanshupandita07

<br/>
The pipeline architecture separates ingestion, storage, retrieval, and generation into discrete, independently scalable layers. PySpark handles distributed transformation on Dataproc Serverless, BigQuery acts as the analytical retrieval layer with optimised partitioning on incident date, and the Gemini API operates as the reasoning engine receiving retrieved rows as structured context. Cloud Scheduler automates the full ingestion cycle at midnight IST, making the system self-maintaining and always current.

<br/>
</details>

<summary><strong>🔷 Pharma Document Intelligence Pipeline</strong> &nbsp;—&nbsp; FDA Adverse Event ETL × BigQuery</summary>
<br/>
An enterprise-grade document intelligence pipeline that ingests FDA adverse event PDFs, applies LLM-driven extraction to parse unstructured semi-structured content, maps outputs to a normalised schema, and loads structured records into BigQuery for analytical consumption. Designed to mirror real-world pharma data engineering workflows, where regulatory documents must be transformed into queryable, auditable datasets at scale.

<br/>
AttributeDetailStackPython · Pandas · BigQuery · Cloud Storage · Gemini API · PDF parsing librariesScale2,106 structured records extracted and loaded to BigQueryPerformanceChunked PDF processing · Schema-validated output · Idempotent load patternsArchitectureCloud Storage (raw PDFs) → Gemini extraction → Pandas normalisation → BigQueryImpactReplicates pharma/life sciences data engineering workflows — directly aligned with ZS Associates and Lloyds Banking Group target rolesRepositorygithub.com/Priyanshupandita07

<br/>
The pipeline treats the Gemini API as an extraction runtime rather than a chat interface — passing structured system prompts with document chunks and enforcing JSON output schema for downstream compatibility. Pandas handles the normalisation and validation layer, and BigQuery receives the final records via the Python client library with merge semantics to ensure idempotent loads.
