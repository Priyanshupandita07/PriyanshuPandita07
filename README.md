<h2>⬡ &nbsp; Featured Projects</h2>
</div>
<br/>
<details>
<summary><strong>🔷 Chicago Crime Intelligence Assistant</strong> &nbsp;—&nbsp; RAG Pipeline on BigQuery × Gemini</summary>
<br/>
A production-grade Retrieval-Augmented Generation system built over 500K+ rows of Chicago crime data hosted in BigQuery. The system accepts natural language queries, retrieves semantically relevant crime records, and synthesises analytical responses using Gemini 1.5. Designed as a domain-specific intelligence layer over a large-scale public dataset, demonstrating how LLMs can be grounded in structured data warehouses for enterprise Q&A applications.
<br/>
Attribute	Detail
Stack	Python · BigQuery · Gemini API · PySpark · Cloud Storage · Dataproc Serverless
Scale	500,000+ crime records · Automated nightly ingestion via Cloud Scheduler
Performance	Serverless Spark execution · Partitioned BigQuery tables · Optimised SQL retrieval
Architecture	RAG loop: query → BigQuery retrieval → Gemini context injection → structured response
Impact	Demonstrates LLM grounding over structured warehouse data — portfolio-differentiating for DE + AI roles
Repository	github.com/Priyanshupandita07
<br/>
The pipeline architecture separates ingestion, storage, retrieval, and generation into discrete, independently scalable layers. PySpark handles distributed transformation on Dataproc Serverless, BigQuery acts as the analytical retrieval layer with optimised partitioning on incident date, and the Gemini API operates as the reasoning engine receiving retrieved rows as structured context. Cloud Scheduler automates the full ingestion cycle at midnight IST, making the system self-maintaining and always current.
<br/>
</details>

<details>
<summary><strong>🔷 Pharma Document Intelligence Pipeline</strong> &nbsp;—&nbsp; FDA Adverse Event ETL × BigQuery</summary>
<br/>
An enterprise-grade document intelligence pipeline that ingests FDA adverse event PDFs, applies LLM-driven extraction to parse unstructured semi-structured content, maps outputs to a normalised schema, and loads structured records into BigQuery for analytical consumption. Designed to mirror real-world pharma data engineering workflows, where regulatory documents must be transformed into queryable, auditable datasets at scale.
<br/>
Attribute	Detail
Stack	Python · Pandas · BigQuery · Cloud Storage · Gemini API · PDF parsing libraries
Scale	2,106 structured records extracted and loaded to BigQuery
Performance	Chunked PDF processing · Schema-validated output · Idempotent load patterns
Architecture	Cloud Storage (raw PDFs) → Gemini extraction → Pandas normalisation → BigQuery
Impact	Replicates pharma/life sciences data engineering workflows — directly aligned with ZS Associates and Lloyds Banking Group target roles
Repository	github.com/Priyanshupandita07
<br/>
The pipeline treats the Gemini API as an extraction runtime rather than a chat interface — passing structured system prompts with document chunks and enforcing JSON output schema for downstream compatibility. Pandas handles the normalisation and validation layer, and BigQuery receives the final records via the Python client library with merge semantics to ensure idempotent loads. The project is scoped to demonstrate pharma-domain data engineering maturity for life sciences hiring teams.
<br/>
</details>

<summary><strong>🔷 Pharma Sales ETL Pipeline</strong> &nbsp;—&nbsp; Cloud-Native Data Engineering on GCP</summary>
<br/>
A complete cloud-native ETL pipeline ingesting pharma sales data from Cloud Storage, transforming and enriching it with Pandas, and loading structured output to BigQuery. Built as a portfolio-grade demonstration of the foundational data engineering stack — source ingestion, schema enforcement, transformation logic, and warehouse loading — with production conventions throughout.
<br/>
Attribute	Detail
Stack	Python · Pandas · BigQuery · Cloud Storage · GCP Python Client Libraries
Scale	2,106 rows with cross-regional and product-line segmentation
Performance	Columnar storage in BigQuery · Efficient batch loading · Type-safe schema enforcement
Findings	Region-level revenue variance, product performance clustering, sales trend extraction
Impact	End-to-end GCP data engineering showcase aligned with enterprise pharma data roles
Repository	github.com/Priyanshupandita07
<br/>
This pipeline establishes the GCP data engineering baseline: raw files in Cloud Storage serve as the landing zone, Pandas provides the transformation layer with explicit dtype enforcement and null handling, and BigQuery receives the output via the official Python client with schema-on-write. The project demonstrates comfort with the full GCP data stack and understanding of data warehouse loading patterns that appear in enterprise DE roles across pharma and banking.
